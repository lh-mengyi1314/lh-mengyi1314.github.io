【学习】Gridea+Github部署个人博客

http://imengyi.ink/zbWUCelAN/
http://imengyi.ink

热度🔥: loading...

引言

主要是因为一些官方博客（知乎、博客园等），你发表一些自己的看法或见解是要先审核的，而机器审核会使得我们原本并不违规的内容被禁，导致自己的博客账号被封，所写的内容也就灰飞烟灭啦。

我自己是从零开始部署的，其实也就三-五天就好啦，主要是一些细节问题，我这里将会讲解我所遇到的一些问题，有的内容是大神写的比较好的，我就直接放链接啦~~~💖💖💖

1、Gridea的安装

我们进入Gridea的官方网站下载安装

2、Gridea与Github的远程连接

🔵创建 GitHub 账号，并创建一个仓库 Token。点击GitHub，进行注册。登录 GitHub 之后，点击这里创建一个 Token，勾选上 repo 的相关权限即可。生成之后记得把 Token 复制到你的本地，因为一旦关闭网页将不能再看到它。

🔵🔵创建仓库，存放构建后的网站文件。创建一个公开仓库，名为用户名.github.io。（将用户名替换为你的 GitHub 用户名）

🔵🔵🔵进入 Gridea 进行远程设置。按照下图的示例进行设置，然后点击「检测远程链接」，进行配置检查。

🔵🔵🔵🔵同步。点击 Gridea 中的 同步按钮即可。初次部署等待几分钟之后就可以去浏览器访问了，默认地址为你在第 3 步所填写的域名。

以上，就完成一个静态博客的搭建了，期待你的每一次落笔成文。

第二个是nodecache添加加速的网站，cname到他给你的地址。
